# カスタムスキルの作成仕様書

gstackでは、「特定のワークフローに特化した新しいAIエージェントの専門知識（スキル）」をプロジェクト内に追加・拡張することができます。このドキュメントでは、そのカスタムスキルを作成するための仕様と手順を解説します。

## 1. 基本となる概念とディレクトリ構造

gstackのスキルは、**「1つのディレクトリ ＝ 1つの専門スキル」**として管理されます。
既存のリポジトリ直下、あるいはAIエージェント用のフォルダ群の中に、新しくフォルダを作成してください。

### 必須ファイル
新しいスキルを作成する際、そのディレクトリ内に必ず用意すべきファイルは **`SKILL.md.tmpl`** のみです。
（※ `SKILL.md` 自体はビルド時に自動生成されるため、手動で直接編集してはいけません。）

```text
my-custom-skill/            <-- スキル名となるディレクトリ
└── SKILL.md.tmpl           <-- 人間が書くプロンプトや手順書（テンプレート）
```

---

## 2. SKILL.md.tmpl の記述様式とプレースホルダー

`SKILL.md.tmpl` は、AIエージェントに対する指示書（プロンプト）です。
ここには、エージェントに「どのような思考プロセスで」「どのようなタスクを実行してほしいか」を人間が記述します。

その際、gstackのアーキテクチャと連携させるために、**必ずいくつかのプレースホルダー（`{{...}}`）を埋め込む必要があります**。

### 必須のプレースホルダー

1. **`{{PREAMBLE}}`**
   - **配置場所:** ドキュメントの最上部（最初）
   - **役割:** セッション管理、自己改善ループの定義、AIエージェントへの「質問フォーマット（AskUserQuestion）」の統一など、gstackを稼働させるためのインフラ的なお約束ごとを自動展開します。

2. **`{{COMMAND_REFERENCE}}`**
   - **配置場所:** 任意の場所（通常は下部）
   - **役割:** gstack本体から抽出された最新の「利用可能なコマンド一覧（`snapshot`, `click`, `goto`など）」を自動で展開します。これにより、コマンドの仕様が変わってもドキュメントが常に最新に保たれます。

### オプションのプレースホルダー
既存のワークフロー（QA、デザイン監査など）の思考法を再利用したい場合に使用できます。

- `{{QA_METHODOLOGY}}` : QAテストの手法や観点を展開
- `{{DESIGN_METHODOLOGY}}` : UI/UXデザイン監査の基準を展開
- `{{TEST_BOOTSTRAP}}` : テストフレームワークの検出・セットアップ手順を展開

### 記述例（SKILL.md.tmpl）

```markdown
{{PREAMBLE}}

# 私のカスタムスキル（My Custom Skill）

あなたはユーザーの代わりにWebサイトを巡回し、特定の情報を収集する専門エージェントです。

## ワークフロー
1. 対象のURLに `goto` コマンドでアクセスする。
2. `snapshot -i` を使って画面内の要素を確認する。
3. 必要な情報を読み取り、ユーザーに `AskUserQuestion` フォーマットで報告する。

## コマンドリファレンス
以下のコマンドを利用してブラウザを操作してください。
{{COMMAND_REFERENCE}}
```

## 3. スキルのビルド（生成）手順

`SKILL.md.tmpl` を作成・編集したら、それをAIエージェントが読み込める `SKILL.md` に変換する必要があります。

リポジトリのルートディレクトリで以下のコマンドを実行してください。

```bash
bun run build
```

（内部的に `bun run gen:skill-docs` が呼び出され、各ディレクトリの `.tmpl` ファイルとソースコードのメタデータがマージされて `SKILL.md` が生成されます。）

## 4. 注意点・ベストプラクティス

1. **`SKILL.md` はコミットする**
   生成された `SKILL.md` はGitの管理下（コミット対象）に含めてください。これにより、AIエージェントがランタイム時にビルド待ちゼロでスキルを読み込めます。
2. **Fail-Fast な手順を書く**
   AIが迷わないよう、「要素が見つからなかったらどうするか」「エラーが出たらどうするか」のフォールバック手順を明確に記述すると、より堅牢なスキルになります。
